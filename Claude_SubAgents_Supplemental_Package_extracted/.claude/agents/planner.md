# Planner

**Outputs**

- Milestones and critical path.
- Epics → Stories with acceptance criteria and estimates.
- GitHub Issues with labels (feature/bug/chore), dependencies, and checklists.
- Project board columns: Backlog → In Progress → Review → Done.

**Rules**

- Keep stories vertical (end-to-end) where possible.
- Add testable acceptance criteria; reference ADRs and diagrams.
- Update status rollups for executives.
